1. **api/**：存放 API 接口定义和路由配置，按业务模块划分不同的路由文件
2. **controller/**：实现 API 接口的处理逻辑，接收请求并返回响应
3. **service/**：包含核心业务逻辑，处理复杂的业务规则和数据处理
4. **model/**：定义数据模型和数据库交互，包括结构体定义和 ORM 映射
5. **middleware/**：存放中间件，如认证授权、日志记录、请求验证等
6. **utils/**：工具函数和通用辅助方法，如加密、日期处理、格式转换等
7. **config/**：配置文件和配置管理，包括环境变量和配置加载
8. **router/**：路由注册和路由管理，集中管理所有路由规则
9. **docs/**：API 文档和项目文档，可使用 Swagger 自动生成
10. **test/**：单元测试和集成测试代码，按模块组织测试文件
11. **static/**：静态资源文件，如图片、CSS、JavaScript 等
12. **logs/**：日志文件存储目录，按日期或级别分类存储日志
